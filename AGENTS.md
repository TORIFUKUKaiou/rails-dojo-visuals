# AGENTS.md

このリポジトリは、Rails Dojo の授業で補助的に使う Visualizer です。
主目的は「初学者がコードの実行順、データの変化、処理の流れを目で追えること」です。

## 最優先の方針

- 派手な動きは歓迎。ただし、何が起きているかを追えなくなる演出は避ける。
- 画面上の動きは、教材のコード理解を助けるためのものにする。
- 対応する教材 Markdown の説明、コード例、用語と食い違わないようにする。
- 特定の週だけに閉じた設計にせず、今後の回や別テーマにも同じ方針を展開できるようにする。

## ステップ表現のルール

- 初期状態では、箱・配列・変数などは薄く表示する。
- 該当するコード行をステップ実行して初めて、対応する箱や配列を濃く表示する。
- 右側のコードハイライト、左側の図、下の説明文、実行結果のタイミングをできるだけ一致させる。
- 実行済みのコード行は中くらいの濃さ、未到達のコード行は薄く、現在行だけを強く表示する。

## 代入の見せ方

- `=` は、初学者には難しいので必要に応じて2ステップに分ける。
- 1ステップ目: 右側を参照する、または計算する。
- 2ステップ目: 左側の変数や配列要素を更新する。
- 例: `foods[1] = "うどん"`
  - まず `foods[1]` の場所を強調し、値はまだ `"ラーメン"` のままにする。
  - 次のステップで `"うどん"` に上書きする。
- 例: `total = total + score`
  - まず `total + score` を計算し、`total` は旧値のままにする。
  - 次のステップで計算結果を `total` に上書きする。
- ただし、`total = 0` や `scores = [...]` のような初期値設定は、冗長になりすぎるため1ステップでよい。

## ループの見せ方

- `each` などのループは初学者が追いにくいので、丁寧に表現する。
- `each` 行で要素を1つ取り出す。
- 次の行で `puts` や計算を実行する。
- その後、また `each` 行に戻るように見せる。
- 「まとめて一気に進む」より、数回は愚直に繰り返す。
- 表示結果は、`puts` のステップで増やす。

## 条件分岐の見せ方

- `if` の行では、まず判定結果を `true` / `false` として見せる。
- その次に、`true` のルートまたは `false` のルートに入る。
- `score` のような一時変数は、判定箱とは別の箱として表示する。
- `score` 箱は初期状態では薄く、`each` で値が入った時点で濃くする。

## UI

- レッスン選択メニューは左端に控えめに置く。
- メイン画面には、対象の学年・回・テーマが分かる表示を残す。
- 授業用なので、1280x720程度の画面でも主要要素が収まるようにする。
- ボタンや文字が重ならないようにする。

## 検証

変更後は最低限、次を実行する。

```bash
npm run lint
npm run build
```

ローカル表示は次で確認する。

```bash
npm run dev
```

表示先:

```text
http://localhost:3000/
```

## 注意

- README やデプロイ設定の整理は別タスク。授業で使える見た目と動きを優先する。
- `node_modules/` と `dist/` はコミットしない。
- `package-lock.json` は依存関係を固定するためコミット対象。
